Mes Traces

Plugin “InfiniteStamina” (REPO, C#)

Conception et développement d’un mod pour le jeu REPO, permettant de donner de la stamina infinie au joueur via BepInEx et Harmony. Le code initialise un singleton, configure le logger et applique des patches runtime sur les méthodes de stamina dans "Awake()".

Savoirs mobilisés :
  • POO en C# : classes, namespaces, constantes, modificateurs d’accès
  • Framework BepInEx : attributs "[BepInPlugin]", héritage de "BaseUnityPlugin"
  • Harmony : création d’un objet "Harmony", utilisation de "PatchAll"
  • Logging : "ManualLogSource", "LogInfo"
Savoir-faire acquis :
  • Mise en place du pattern singleton pour un plugin Unity
  • Organisation modulaire d’un projet de modding
  • Application de patches dynamiques sans modifier les DLL d’origine
  • Structuration de messages de log clairs et traçables
Points à améliorer :
  • Pas de gestion de configuration utilisateur (fichier JSON ou GUI)
  • Absence de tests unitaires ou d’intégration pour valider chaque patch
  • Logs peu détaillés (manque de niveaux DEBUG/ERROR)
  • Couplage fort entre Harmony et la logique métier (difficile à découpler)

Administration des commentaires (Flask, Python)

Développement d’un Blueprint Flask ("admin_commentaire") pour gérer l’affichage et la suppression de commentaires d’articles. Routes GET/POST, récupération sécurisée des paramètres, exécution de requêtes SQL et rendu via Jinja2.

Savoirs mobilisés :
  • Flask : Blueprints, routes, "request", "render_template"
  • Gestion de base SQLite : curseur, transactions, "commit()"
  • SQL : requêtes paramétrées, prévention d’injection
  • Jinja2 : boucles et affichage dynamique de données
Savoir-faire acquis :
  • Modularisation du back-end avec Blueprints
  • Gestion des flux CRUD (GET list, POST delete)
  • Sécurisation et validation des inputs utilisateur
  • Intégration front-end via templates Jinja2
Points à améliorer :
  • Pas de mécanisme de pagination pour les listes longues
  • Absence de gestion fine des erreurs (404, 500, rollback)
  • Pas de protection CSRF sur les formulaires POST
  • Templates statiques sans composant réutilisable (duplication de code)
